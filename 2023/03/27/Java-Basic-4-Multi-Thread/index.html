4 Multi-Thread

What is Thread ?

**A thread is the smallest unit of processing that can be scheduled by an operating system. **

Difference Between Thread and process ?

A process is an instance of a computer program that is being executed by one or many threads

How to Implement Thread in Java?

  • Create a thread extends Thread and overrides RUN method
  • Create a runnable class implements Runnable and overrides RUN method
  • myThread.start() / new Thread(myRunnable).start()

Understandings:

  • thread starting doesn’t mean the thread really “run” over its code
  • think of thread is kind of “lazy”, switch between all these threads – “fake multi-threads”
  • time slicing == slice the time arranged by OS, when there is a new thread, thread scheduler tells CPU there exists a new thread

Synchronized

is a keyword, can be added in front of code block and function, when a method or block of code is marked as synchronized, only one thread can execute it at a time.

wait

**The **wait() method is used to pause the execution of a thread and release the lock on the object until it is notified by another thread.

Understanding:

current thread gives up its right and current thread gets into waiting room

Code for current thread stucks

notify

**wake up a thread that is waiting on the same object. **

Understanding:

current thread notify others in waiting room to queue for the entrance of the “lock room”

Very Interesting Example of Hospital.java example

When the patient with higher id gets into this treatment thread later, it will cause a problem.

Example: 1,2,6,8,9

1 first and then finish

2 then and then finish

6 gets in and get stucked in waiting room

8 gets in and get stucked in waiting room

9 gets in and get stucked in waiting room

6, 8, 9 are trapped into waiting room and nobody call them to get into queue

in this secerio use wait(time) may be better

finalize

why protected ?

The only time you want to call finalize is gc. So protected makes this method can be only called in gc not other times.

DeadLock

2 or more threads holding each others resource without releasing

How to Change this scenario?

use tryLock to reduce the effect of synchronized

can use Future to receive the value for the output of the value

queue has a size, if the size of queue exceeds a number, producer thread will be blocked

if the size is 0, consumer thread will be blocked.
