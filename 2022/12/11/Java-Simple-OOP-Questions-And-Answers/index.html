Java Simple OOP Questions And Answers

Created2022-12-11|Updated2022-12-12|Core Java Learning
|Post View:
1 JRE, JVM and JDK
  1. JRE
  • JRE is Java Runtime Environment.
  • It is a package of everything necessary to run a “COMPILED” java program.
  • JRE includes Java Virtual Environment(JVM) and Java Class Library and other stuffs.(Needs to be modified later).
  1. JDK
  • JDK is Java Developer Kit
  • JDK has everything JRE has with compiler(javac) and tools.
  • JDK can complie a .java file to .class file and then all other stuff turns to JRE.
  1. JVM
  • JVM is Java Virtual Machine.
  • JVM converts .class file to binary code and then Java ByteCode is implemented by OS.
  1. Relationship
  • JDK == JRE + tools + compiler == JVM + Java Class Library + … + tools + compiler
2 OOP and OOP’s principle
1 What is OOP?

OOP refers to Object-Oriented Programming. It uses object as a primary source to implement programming.

2 OOP’s principle
  1. Abstraction
  • Object should only expose a high-level mechanism. This mechanism should hide internal implementation.
  • “Using phone” only knows about button with internal implementation.
  1. Encapsulation
  • Object keeps its state private. And State can be changed only by a list of public methods.
  • the state can be seen as “variables”.
  1. Inheritance
  • We can create a (child) class by deriving from another (parent) class to reuse the common logic and extract the unique logic.
  1. Polymorphism
  • Polymorphism gives a way to use a class exactly like its parents.
  • But child keeps its own methods as they are.
  • “many shapes” in Greek
  • member variable using “father”, member method using “son” -> Different “shape”
3 Basic Idea
  1. Class and Object
  • Class is a blueprint or a set of instructions to build a specific type of object.
  • Object is an instance of Class. It consists of methods and properties to make it useful.
4 Constructor
  1. Definition
  • Constructor, like a method, used to create an object and set the initial state of the object.
  • Return type as a Class
  1. Properties
  • If we don’t implement in code, we will have a default one without args.
  • If we implemented constructor in code, we can only use that.
  • We can implemented many constructors with different args.
  • It cannot be inherited and constructors cannot be final, static and abstract.
  • Cannot be final, static and abstract.
5 static - need to be supplemented
  • static is used to show this attributes to a Class not only a object.
  • Main method needs to be static since JVM can call it to start the program (JVM loads the full class and implements the method)
  • In this way, static methods cannot use non-staic method or non-staic member methods.
  • static method cannot use super() method since super() used as a reference to the object of the superclass.
6 overload and override
  • overload means multiple functions names with same name but different arguments.
  • override means a new implementation of a method with the same name in the subclass.
  • Cannot override static method since overrided method is decided at “RUN-TIME” level to have the same name.
7 Abstrac Class and Interface
  • Abstract Class has abstract methods(may be not)
  • Abstract methods have only function prototype but not specific implementations
  • cannot have instance/needs to be inherited by other class(abstract class and non-abstract class)
  • Interface contains the method needs to be implemented and constant variables.
8 final
  • used for variable: cannot assign a new value
  • used for method: cannot be overrided
  • used for class: cannot be inherited
