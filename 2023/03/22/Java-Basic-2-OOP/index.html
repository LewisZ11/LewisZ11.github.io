static vs non-static

What is Static ?

static is a keyword. It can be added in front of field, function, inner class and block, all available element to make it class level.

Followings are understanding:

Static block

  • static block is going after static field variable
  • static block can assign a value of static variable

Static inner class

Static Inner Class
  • 理解静态只能访问静态，所以静态类内部不能有一些非静态的东西
  • 静态类是对外面来说的，所以在外部类的静态方法中或者是静态field，直接使用new 方法创建调用就可以了
  • 从其他类中访问，可以直接用 类名.new 内部类名();
Non-Static Inner Class
  • 里面不能有静态的东西
  • 访问其他静态资源时，直接访问就可以了
  • 可以访问其他非静态资源，用 new 内部类() / new 外部类().new 内部类()
example

Another class:

static method

can be used in Interface, use Interface_Name.method_name works

  • Static Methods are class level, in jvm it belongs to class level information.
  • Before run time, we need to load class, so class level information are loaded.
  • When non-static method is called, there must be a object.
Why Main Method is static ?
  • If Main method is not static, it means main method is object-level, which meaning that there must be a project.
  • In the very beginning you need to run main method, but at that time jvm doesn’t have any objects.
  • Obviously it is controversial. So Main Method is static.
Why static method only calls static method?
  • Static already loaded before run time.
  • you can only call function from object but we don’t find the object at very beginning.
  • but you can create a object and call it.
Can we call static method in non-static method?

Yes.

  • **calling Non-static function must have a object **
  • when non-static function is called, static function must exist since already loaded
Can we can non-static method in non-static method?

Yes.

  • Object-level function is available in object be available at same time.

Utils Class

  1. just use static method
  2. with private constructor since we don’t want to use its object, just to use its method.

OverLoad

  1. overload = same function name with different params, maybe return type is different
  2. sout is a example of overload

Java Passing By Value / Reference?

Java is passing by the value of reference

Understanding:

  • Java is passing by reference when for non-primitive/reference type
  • Java is passing by the value when for primitive type

Primitive Types

types :

  • byte = 8 bit, boolean 我们只需要1 bit代表, 但是用1 byte便于java 程序控制，有些boolean 在其他JVM中不是一个byte
  • Short char 2 byte
  • Int float 4 byte
  • Long double 8 byte

default value For All types:

  • **1 bit = 0 / 1 **
  • boolean false
  • double = 0.0
  • String s;
  • 变量赋引用的值之前，先赋上0，如果要打印地址显示为null

JVM地址是用int表示

  • **电脑/OS大多64位/32位，这个是啥意思？ **
    • computer 64 bit = hardware in address is a 64 bit number
    • Windows 64 bit = OS also allows hardware address in 64 bit
  • JVM 是 32 位的, 说明操作32位地址，所用显然地址是int

Java bean - POJO bean - POJO

Plain Old Java Object

  • In industry, we seperate data with logic
  • 方法类基本上属于业务逻辑，放到service里面会更好
  • 注意getter和setter这种方法, 保持规定逻辑不要太复杂，
  • 如果在set和get加入其他逻辑，程序可以跑起来，但是对以后使用框架开发（Spring）不利，所以遵守规则

Constructor : function

  1. same name as class name
  2. no return type
  3. Public / private
  4. object level function
  5. By default, if not constructor is provider manually, default constructor exist
  6. new keyword, will call constructor once the object is created, to initilize
  7. for each object, constructor can only be called once after its creation
  8. constructor can be overloaded, cannot be overridden
  9. when new is calling constructor, it will call it’s parent constructor by default
  10. super() is by default as the 1st line of each constructor. Super 表示parent的object not class

首先明白多少个constructor for each class，然后this 和 super 必须放在第一行，要不就不写

Object and Hashcode

  • Java.lang.Object Top level class in java / every other class in java extends Object
  • **toString() 改写成 field的information **
  • toString()
    • tohexString() => 16进制
    • hashCode() => location, toString() 可以改写成field information of object, 方便理解

继承问题

  • is Java support multi extend ? 1 class ext from 2 classes No
  • **Explain java diamond issue **
  • 1 java class implements multi interfaces ? Yes
    • 如果多interface 函数名相同是不是也可以，对可以，因为你只是写了函数名 但是没写函数实现
  • interface implement other interface No
  • interface can extend other interface Yes

Reflection

basic : from class create an object / from object to find class

**supposed to use reflection **

normally doesn’t use a reflection

use it in framework

不同修饰符对应的方法

  • public function can be called from anywhere
  • protected function can be called some same package
  • package function can only be called from some package or sub classes
  • private function can only be called with same outer class

继承状态下的问题

  • public function can be overriden from anywhere
  • protected function can be overriden from anywhere
  • package function can only be overriden from same package
  • private function cannot be overriden

Encapsulation

Encapsulation is a way to restrict the direct access to some components of an object, so users cannot directly access state values of a particular object.

Understanding: “has a” relationship

Inheritance

to inherit attributes and methods from one class to another in java.

Understanding:

  1. static function cannot be overriden
    **(1) But If **
    father and son both have same name static method + polymorphism
    看左边指向是哪个类，就调用哪个类的方法

    (2) 其他就是正常情况了

  2. private method cannot be overriden

  3. use @Override annotation

    1. 1.5 开始才有annotation, no-logic here, 包括两个注解
    2. @Target(ElementType.METHOD) can be put top of method
    3. @Retention(RetentionPolicy.SOURCE) Checking works at compile time

  4. if a function cannot be found from an object, then search from it’s reference class, if still not found, keep search from its parent class

Abstraction

it is the process of hiding certain details and showing only essential information to the user in java.

Understanding

  1. meaning everything in blueprint but without implementation
    How do you tell other about the class in blueprint ? using abstract keyword.
    But If actually everything is really implemented, You can still tell other it is in blueprint
    In this way, we can understand some basic ideas of abstract class - It can create an object since it is in blueprint not finished
    Also abstract class may don’t have abstract method, all of its method can be fully implemented. But still you cannot create it.
  2. Abstract class can be implemented by sub class, can extend other abstract class but without fully implementation, can implement father’s abstract methods.

Polymorphism

Subclasses of a class can define their own unique behaviors and yet share some of the same functionality of the parent class.

Understanding
  • multiple shapes
  • **change the shape easily **
  • but the original object is not changed
UpCasting and DownCasting
  • UpCasting: 就是多态, convert sub implementation to a father object reference
  • DownCasting: convert father object to sub object, but the actual implementation for this object should have sub implementation
Notice
  1. take care of the exact object created in heap
  2. shape can be changed with itself, father, grandfather
  3. Directly downcasting is not allowed
