Javascript

Variable

  • 变量命名规则
  • 所有的声明变量（但不赋值）都需要用var，例如 var x; 这句话
  • 用var时可多次不同赋值，甚至可以赋值不同的数据类型
  • var 定义变量，如果没有初始化就定义为undefined，例如 var x；
  • var 定义的变量在函数作用域中，也就是说这个方法可以在外部调用
  • const 作为常量，不可以修改，必须再声明时初始化 例如 const x = 100；
  • 但是const用作数组时不一样，不能为数组重新赋值一个新数组，但是可以为数组来一些操作
  • let 块级作用域，也可以理解为局部变量，let不能变量提升
  • 变量提升就是把声明var和function() “放到前面”，这样可以直接使用，比如说赋值操作之类。。。

Boolean

比较运算符

  • 第一种是 ==比较，它会自动转换数据类型再比较，很多时候，会得到非常诡异的结果；
  • 第二种是 ===比较，它不会自动转换数据类型，如果数据类型不一致，返回 false，如果一致，再比较。

JavaScript的设计者希望用 null表示一个空的值，而 undefined表示值未定义。

事实证明，这并没有什么卵用，区分两者的意义不大。大多数情况下，我们都应该用 null。undefined仅仅在判断函数参数是否传递的情况下有用。

Object Tranversal

四种方式

Array Method

toString()
  • 将arr拼接成字符串，不要求arr里的元素必须是字符串类型
join()
  • 和toString()方法非常相似，但是注意join括号内的字符串用法
pop()/shift()
  • pop() 出来的元素可以用let/const/var 接收
  • Shift() 取出第一个元素
  • Null/undefined 也接收
  • 数组为空接受的是undefined
push()/unshift()
  • 如果push为空，return为新数组长度，等于原数组长度，数组没什么变化
  • push为null/undefined, 直接加入元素
change elements
  • 直接change 下标对应的值， 从0开始
  • 可以扩容数组的长度
  • 如果长度不一致，相当于扩容的是undefined
delete

delete某一项元素时，把该元素变成undefined，不推荐使用

concat()
  • Concat() merges exisiting arrays
  • Concat() 可以有多个参数（多个数组一起合并），或者是直接合并一个元素
splice()

**The first parameter (2) defines the position **where new elements should be added (spliced in).

**The second parameter (0) defines **how many elements should be removed.

**The rest of the parameters (“Lemon” , “Kiwi”) define the new elements to be **added.

slice()
  • 切片形成新的arr，旧arr不变
  • 如果slice（number） 数目大于等于length，切出来的是null
  • 如果slice（number）小于0，但是其绝对值小于等于length，倒着切
  • 如果slice（number）小于0，且绝对值大雨length，整体直接返回，想想倒着切，把所有都切到了
sort()
callback
  • forEach/Map 方法中调用
  • **The **map() method creates a new array by performing a function on each array element.
  • **The **map() method does not execute the function for array elements without values.
  • **The **map() method does not change the original array.
filter()

**The **filter() method creates a new array with array elements that pass a test.

reduce()
  • **The **reduceRight() method does not reduce the original array.
  • reduce/reduceRight 方向不同罢了

CSS Model

Box Model

The box model is used to determine the height and width of the rectangular box. The CSS Box consists of Width and height (or in the absence of that, default values and the content inside), padding, borders, margin.

  • Content: Actual Content of the box where the text or image is placed.
  • Padding: Area surrounding the content (Space between the border and content).
  • Border: Area surrounding the padding.
  • Margin: Area surrounding the border.

Margin

  • Margin: 4 values top/right/bottom/left
  • 3 values: top/(right+left)/bottom
  • 2 values: top+bottom/left/right
  • Value: 上下左右

Padding

  • 和Margin一致，但是width包括padding，所以有时计算要记得扣除

Border

  • Border: width style color
  • Border-style/-width/-color: 如果四个值/三个值/两个值和Margin规则一样

Position

  • Static: s not positioned in any special way; it is always positioned according to the normal flow of the page
  • An element with position: relative; is positioned relative to its normal position:
  • **An element with **position: fixed; is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position the element
  • Absolute is positioned relative to the nearest positioned ancestor
  • **An element with **position: sticky; is positioned based on the user’s scroll position.

DOM

  • Find elements First
  • Finding HTML elements by id
  • Finding HTML elements by tag name
  • Finding HTML elements by class name
  • Finding HTML elements by CSS selectors
  • Finding HTML elements by HTML object collections

**The **HTML DOM is an Object Model for HTML. It defines:

  • **HTML elements as **objects
  • Properties for all HTML elements
  • Methods for all HTML elements
  • Events for all HTML elements

FORM

Label
  • For用法，主要用来绑定后面的输入input，当点击label之后就可以直接把input光标打开
Input
  • type：输入方式
  • value: initial value for an input field
  • name 属性规定 input 元素的名称，name 属性用于对提交到服务器后的表单数据进行标识

form action：完成表单后提交到哪里 method：提交方式
