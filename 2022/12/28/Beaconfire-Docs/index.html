BFCM Project

Introduction

**a huge amount of customer shopping on website within a very short period of time on prime day in Amazon. The buying actions for these customers will cause a big pressure on backend system. This project is aimed to build a system that can face the pressure of high concurrency and some functionalities. **

  • The customer can buy an item, when the stock count of this item is greater than zero
  • The customer can only buy one item for every kind of product
Stock Count Question

Logic: 查Redis库存 -》库存OK就减一 -〉生成订单

  • 把查Redis库存和库存满足条件就减一这个操作用lua.script 封装起来，变成一个原子性操作
  • 在生成订单的过程中判断一下MySQL数据库下面该产品的库存在自减一的之后会不会大于0，如果库存之后是小于0，就说明不能生成订单
Duplicate Orders
  • 在生成订单之后，我们可以把用户和产品id绑在一起作为key，把生成的order作为value
  • key-value pair can be stored in Redis Template
  • Next time we can check the current userid and product id before checking the stock count. If we find the value is not empty, we can directly return fail information
  • This can help the customer to buy only one item for every kind of product and cannot generate duplicate orders.
Small Optimization
  • After Checking the MySQL database, we can just use the message queue to send the order message and then use receiver to receive the message and generate the order.
  • Use message queues to buffer instantaneous traffic, convert synchronous direct calls into asynchronous indirect pushes, use a queue in the middle to undertake instantaneous traffic peaks at one end, and push messages out smoothly at the other end
  • We can also use a in-memory hash map to store a “out of stock” information, if this product is out of order, we can just update it to a hash map by putting key as this product name and value as true.
  • So next time we can check this in-memory hashmap first, if map.get(productName) return true. We can directly return out of stock error and it saves time and pressure for checking redis database.

Ads Project

Introduction

This system mainly contains two subsystems: advertisement sponsor system and advertisement search system.

The advertisers put specific advertisements or we can call it creative to advertisement sponsor systems. Each advertisement binds with some like advertised product name, advertised type and so on.. This sponsor systems store all of these binded advertisement.

Then we can use search system with some keywords to search all ads in sponsor systems.

ex: The BMW advertisers can put a BMW 530 car advertisement in sponsor system with different keywords like carname(BWM ) and adtype(car). We can use a advertisement search system to search car, then we can find this advertisements. …

Index Question
  • We can create index to help us to search desired ads.
  • We know that each advertisement binding with several keywords. Index is a mapping from keyword to desired ads.
  • When the sponsor system mysql database changes, we can use a binarylogclient to keep track of database changing and use this log information to update our index. Instead of creating index from full database, updating index helpes us to save a lot of time.
  • KafKa is used to send MySQL binlog information and we use a kafka listener to use this binlog information to update index.

Exception Concept

Exception Handling

Java Provides an Exception Handling mechanism to handle RUNTIME errors that occur in JVM.

Checked/Unchecked

  • Checked Exceptions check at compile time in Java.
  • Unchecked Exceptions happen at RUNTIME. They are not checked at compile time.

try catch finally block

Execute try block -> when exception occurs, catched exception in catch block matches the block -> execute catch block -> execute finally block

throw and throws

  • Throw: to throw an exception from a method or a static block.
  • Throws: mention the exception thrown by a method in its declaration

When an exception is thrown, it needs to be handled. (try catch finally block)

Exception Propagation

**Let say an exception propagates from one method to another method. **

**A() calls B(), which calls C(), which calls D(). **

**And if D() throws an exception, the exception will propagate from D to C to B to A, **

unless one of the methods catches the exception.

MySQL and NoSQL

MYSQL Info

MySQL is not a programming language. Instead, it is a relational database management system (RDBMS). It is used to store data, not to write programs. The SQL programming language can be used to program a MySQL database.

Relational database are based on the relational model, an intuitive, straightforward way of representing data in tables. In a relational database, each row in the table is a record with a unique ID called the key. The columns of the table hold attributes of the data, and each record usually has a value for each attribute, making it easy to establish the relationships among data points.

A primary key is used to ensure data in the specific column is unique.

A foreign key is a column or group of columns in a relational database table that provides a link between data in two tables.

Transaction

**A transaction in MySQL is a **sequential group of statements， queries, or operations to perform as a one single work unit that can be committed or rolled back.

Atomicity：all statements or operations within the transaction unit must be executed successfully

Consistency：the database changes state only when a transaction will be committed successfully

isolation：each operation in the transaction unit operated independently

Durability：the result of committed transactions persists permanently even if the system crashes or failed（写磁盘）

NoSQL

NoSQL databases (aka “not only SQL”) are non-tabular databases and store data differently than relational tables. NoSQL databases come in a variety of types based on their data model. The main types are document, key-value, wide-column, and graph.

Spring Framework

The Spring Framework provides a comprehensive programming and configuration model for modern Java-based enterprise applications

IoC: Inversion of control. It creates the objects, configures and assembles their dependencies, manages their entire life cycle. The Container uses Dependency Injection(DI) to manage the components that make up the application. Since the Controlling of Java objects and their lifecycle is not done by the developers, hence the name Inversion Of Control.

**AOP breaks the program logic into distinct parts (called concerns). It is used to increase modularity by **cross-cutting concerns.
