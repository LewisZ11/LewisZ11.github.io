Self-Introduction

Hi, first of all I want to say that it’s my pleasure to be here today and participating in the interview; also super appreciate you taking the time to talk to me. My name is Yibo Zhang. I’m a graduate student studying at University of Southern California. I have a Software Engineering Internship at Junlian Automotive Electronics Co Ltd in china, where I was responsible for implementing company employee info API and display the warehouse inbound and outbound table in frontend. I also have another internship at Let’s Level Up company and mainly responsible for designing some **chatting, journal, user profile pages in frontend. I have some project experiences including a stock search website and a e-commerce high performance order system and a subway retrieval system. I am familiar with many frontend technologies such as Angular, javascript, html, css and some backend technologies like node.js, flask, springBoot. In new chapter for my life, I really want to apply what I learned in school to industry. Although I am not that experienced, I am excited about what I am going to learn through work.

Behavior Question

Tell me your most challenging project/ a time you solved a complex problem?

  • When I was trying to design a stock search website, I need to implement a feature which simulates people selling and buying stocks, which arises a problem for me.

  • I need to keep track of stocks bought when you need to sell your own stocks since obviously you cannot sell more stocks than your own. But it is not required for me to use some backend databases to keep track of stocks owned for each user.

  • So the problem is that how I can keep track of stocks bought with the help of frontend technologies rather than backend database ?
  • So I just thought about how to solve this problem by using frontend technologies rather than backend? ** took a reference on session but I found sessionStorage data gets cleared when the page session ends — that is, when the page is closed which means it is not appropriate if I want to keep the value stored longer in browser. I asked my friend in industry that how can I use web/browser technologies storage functionality without expiration date? My friend told me that I can use localStorage technologies. localStorage technology is kind of similar with sessionStorage but local storage doesn’t have an expired time. **
  • The result is that I used localStorage to keep track of numbers of bought stocks and it works after simulating buying and selling stocks.** **

  • In one of my previous project our team tried to build a CRM project to display company’s employee information and I was responsible for role restriction which is giving different role in a company different access to this system. However, I do not have any previous knowledge on how to do this and I also have a very tight deadline as well.** **
  • So the task for me is to implement role restriction in a very short period of time.
  • I have some discussions with my team leader and he told me he doesn’t know what technology can be used to solve this problem.** **
  • In this project different actions in frontend means different http request in backend. It is very intuitive to think about that we can check the role when http request comes in and we don’t give them access if the role is not match.** **So I start to look for stack overflow and google doc on role based authorization for http request and I found that I can use spring security. Spring Security in the web tier (for UIs and HTTP back ends) is based on Servlet Filters. I tried to use spring security to “filter” different request URLs. I give the admin role in this system to delete company employee’s information right by giving them the access to request “employees/delete” url in backend. ** **
  • As a result, we handed this problem based on learning spring security before the deadline. I received special thanks from my teammates as they appreciate what I was done.

Tell me about a time when you found a simple solution to a complex problem? (Invent and Simplify)

  • In my previous internship, I was in charge of sorting company employee’s information based on one of there attributes like name, role, ascending and descending etc.
  • So the task for me is to write a sorting method for company employee’s information.
  • It is intuitive to think up with a solution which is just writing sorting algorithm for each attributes like name, role. I don’t think this is a great method since you need to write a lot of methods for a lot of attributes. This method will take a long time. In this way, I thought could I do something easier and simpler. Our company uses spring data spa as our ORM framework. Then I tried to look up some official document for spring data rest.I use Spring Data’s method derivation, whereby the query is generated from the method name and signature. Only I need to do is just write method name and signature in an interface without the need to fully implementing it.
  • As a result, I use spring data spa to solve my sorting issues. Spring data spa greatly reduces me from writing a lot of code and checking bugs for what I have written. I achieve my request before deadline and I received positive feedbacks from my colleagues.
